The Gametools Java Library

Gametools is a lightweight and simple to use library for creating 2D games in java. Some features of this library include:

  • Sprites
  • Image and spritesheet loading
  • Collision handling
  • Keyboard and mouse input
  • Image rotation and manipulation
  • Minimal setup code
  • Sprite grouping and automation
  • Sound support
  • Animation

Guide

Setting Up the Library

To begin, download the latest version of the library JAR. This file contains the compiled source code of the library, and must be added to your project. The way this is done depends on the IDE you are using for coding, however if an option exists to attach javadoc from a URL, then feel free to link it to the library javadoc at http://daniel-oliynyk.github.io/GameTools/javadoc/.

Code Layout

Near the top of each file that you plan to use the library in, you should include these two lines:

import gametools.*;
import static gametools.Tools.*;

The first line imports the main components of the library into your project, and second statically imports all the methods from the Tools class, which shortens the amount of code you have to type when using them (typing pt(2, 4) is much simpler than typing out Tools.pt(2, 4) every time).

The main class of your project should extend the Game class. Here's an example:

package sample;

//These two imports are recommended for easier use of library
import gametools.*;
import static gametools.Tools.*;

//Your main class has to extend the abstract game class
public class Sample extends Game {

    public static void main(String[] args) {
        //The main java method that java always runs first
        initialize(Sample.class); //Sets up the tool class to find the base of the project
        new Sample(); //Need to include this for game to run
    }

    @Override
    protected void window() {
        //Method runs before window is created, used for changing game settings
    }

    @Override
    protected void setup() {
        //Method runs once after window is created, used for setup and initialization
    }

    @Override
    protected void run() {
        //Method runs at a set FPS and contains main game code
    }
}

Importing Images

It is very simple to import images with the Gametools library. After the library's Tools class is imported and initialized, images can be loaded by typing:

BufferedImage car = loadImage("racecar.png"); //The race car image is stored in the project root
BufferedImage dog = loadImage("animals/dog.jpg"); //Slashes separate filenames and folders

The root folder of the project is the folder that the class passed into the initialize method of the Tools class. Ideally, images should be imported in the setup() method of the project to prevent performance issues from constantly importing the same image.

Importing Spritesheets

A spritesheet is an image that contains a bunch of smaller images packed together inside it. Storing related images in a spritesheet is a good habit as loading them is quicker than loading images individually. Spritesheets are imported very similarly to regular images in the library:

//Extracts an array of 64 x 64 sprite images from the player spritesheet
BufferedImage[] all = loadSpriteSheet("player.png", dm(64, 64));
//Returns a cropped version of the full set of player sprites going from 4 to 8
BufferedImage[] walk = trimSpriteSheet(all, 4, 8);
//Loads the full explosion spritesheet then returns a trimmed spritesheet going from 12 to 24
BufferedImage[] explosion = loadSpriteSheet("img/explosion.png", dm(16, 16), 12, 24);

It is important to note that all spritesheet methods return BufferedImage arrays and only work if all the sprite images have the same dimensions. Also, the dm(w, h) is a method from the Tools class that is simply a short form for typing new Dimension(w, h).

Drawing Shapes and Text

In the Gametools library, it is possible to draw simple shapes and text on the screen using the painter() object. This painter is a pointer to a Graphics2D object (see a guide on graphics in java), which has built in methods for drawing many things. For example, a red square labelled 'Hello World!' can be drawn by typing:

painter().setColor(Color.RED); //The color of the painter must be set before drawing
painter().fillRect(100, 100, 300, 300); //Draws a rectangle from (100, 100) to (300, 300)
painter().setColor(Color.WHITE); //The color must be changed before drawing white text
painter().drawString("Hello World!", 180, 140); //Writes 'Hello World!' at (180, 140)

For a full list of java graphics methods go to the Graphics2D documentation page.
